Executive Summary

Recommendation: build a small local-first CLI wrapper, but keep it in two layers.

MVP layer

Use targeted YouTube Analytics API pulls plus YouTube Data API enrichment. This covers current youtube-video-audit --owner-analytics <path> needs for views, watch time, average view duration, average view percentage, traffic source, device, geography, subscriber gained/lost counts, daily series, and retention where the Analytics API exposes them.

Scheduled add-on

Add YouTube Reporting API jobs only after the MVP works, mainly for daily bulk history and thumbnail reach metrics such as video_thumbnail_impressions and video_thumbnail_impressions_ctr.

Non-goal

Do not mirror YouTube Studio wholesale. Produce auditable local files with raw API evidence, normalized summaries, field provenance, and explicit gaps[] entries.

Current state: this page converts the existing research brief into an approval surface. It does not create credentials, scripts, OAuth apps, schedulers, private analytics files, or skill changes.

Section Feedback

Official API Findings

OAuth model

Use OAuth 2.0, not service accounts. YouTube Data API documentation says service accounts are not supported for YouTube accounts and fail with NoLinkedYouTubeAccount.

Minimum scopes

Use yt-analytics.readonly and youtube.readonly. The latter is required by the current reports.query reference and also covers read-only YouTube account access.

Avoid in MVP

Avoid broad youtube, monetary, upload, and write scopes unless a later requirement explicitly calls for them.

Refresh-token risk

Refresh tokens can stop working after revocation, six months of disuse, account/token limits, time-based access expiry, admin policies, or external/testing app expiry.

AreaFindingDesign impact
Data API quotaEvery request costs at least one quota point. Default quota is 10,000 units/day for most endpoints; videos.list and commentThreads.list cost 1 unit each.Single-channel risk is low if the CLI batches and avoids repeated searches.
Analytics API quotaEach reports.query request counts as one unit of Analytics API quota; limits are visible in the Google API Console.Batch video filters or group queries where supported; avoid one query per video per metric family when possible.
Data delayendDate is not a promise that all requested days are present. Responses include data only through the last date for which all requested metrics are available.Write data-availability metadata and mark newest windows as potentially stale.
Privacy thresholdsCountry, traffic source details, search terms, external URLs, demographics, and similar dimensions can be omitted, collapsed, or differ from totals.Represent suppression as gaps[] or limited rows, not silent omission.
Reporting API retentionReports cover one 24-hour Pacific-time period, update daily, and are available for 60 days. New-job historical reports cover only the prior 30 days and are available for 30 days.Reporting API is useful as a standing collector, not as the only backfill mechanism.
Backfills and deletesReporting API can replace previous data; deleted videos can create aggregate/per-video mismatches.Use report createTime as an authority signal and preserve mismatch explanations.

Recommended Local-First Architecture

yt-owner CLI
  auth module
    browser OAuth flow for one channel owner
    token refresh and revoke handling
  collectors
    data-api collector: videos.list, commentThreads.list
    analytics collector: reports.query targeted pulls
    reporting collector: optional jobs/reports/download CSV
  normalizer
    maps API names to current skill-friendly field aliases
    writes provenance and gaps
  local store
    secrets outside repo or in gitignored local paths
    raw API responses separated from normalized summaries
    agent-readable summaries under research/youtube/data/
Manual auth

The owner runs yt-owner auth login --channel <slug>. Browser OAuth happens outside agent visibility.

Private tokens

Token material lives outside the repo by default with 0600 permissions and optional macOS Keychain support.

Separate raw evidence

Collectors write raw API evidence separately from normalized summaries and manifests.

Agent-safe summaries

Agents receive normalized paths and raw evidence paths, never credential paths.

Why this shape: it honors the existing YouTube pack's public-first contract, avoids logged-in Studio scraping and screenshots, supports one-off audit pulls without waiting for Reporting API jobs, and leaves a path for scheduled reach/CTR collection later.

Proposed CLI Surface

These are proposed command shapes, not implemented commands.

# One-time owner bootstrap.
yt-owner auth login --channel weeklysota --scopes analytics-readonly,youtube-readonly
yt-owner auth status --channel weeklysota
yt-owner auth revoke --channel weeklysota

# Pull one video for an audit date range and print the owner-analytics path.
yt-owner pull video VIDEO_ID \
  --channel weeklysota \
  --start 2026-06-01 \
  --end 2026-06-30 \
  --include core,traffic,device,geo,retention,reach \
  --emit owner-analytics

# Pull channel-level rollups for metrics review.
yt-owner pull channel weeklysota \
  --start 2026-06-01 \
  --end 2026-06-30 \
  --dimensions day,video \
  --include core,traffic,device,geo

# Refresh recent windows before an audit.
yt-owner refresh latest --channel weeklysota --days 30
yt-owner refresh recent-videos --channel weeklysota --published-after 2026-06-01 --days-since-publish 14

# Optional Reporting API setup and harvest.
yt-owner reporting list-types --channel weeklysota
yt-owner reporting ensure-jobs --channel weeklysota --reports channel_basic_a3,channel_traffic_source_a3,channel_device_os_a3,channel_reach_basic_a1
yt-owner reporting harvest --channel weeklysota --created-after last-success

# Diagnostics for agent-safe use.
yt-owner doctor --channel weeklysota
yt-owner paths --channel weeklysota --video VIDEO_ID
Failure classExpected behavior
Expired or revoked tokenReturn auth_required, keep old normalized files, write a failed-run manifest, and print the login command.
Quota exceededReturn quota_exceeded, leave previous successful snapshots marked stale but usable, and use bounded backoff only.
Metric unavailableWrite a gaps[] entry with source, requested field, reason, and fallback.
Private/deleted/unlisted mismatchWrite a gap with the target ID, API status, and whether public metadata still exists.

Security Model, Consent, And Recovery

Secrets

Store OAuth clients and tokens outside the repo by default. If repo-local storage is ever supported, require a gitignored path such as .local/youtube-owner-analytics/ and file mode 0600.

Consent

For one owner/channel, keep the OAuth app local/testing and accept periodic reauth if external testing refresh tokens expire after seven days.

Verification if shared

If shared beyond the owner account, prepare for Google verification: narrow scopes, accurate consent screen, privacy policy, domain ownership, scope justification, and demo video.

Recovery

yt-owner auth status should test token refresh and a lightweight read query; auth failures write auth_required manifests.

ScenarioExpected behavior
Expired/revoked tokenStop before collection, write failed-run manifest, return auth_required, point to login.
API quota exceededStop current run, preserve previous artifacts, mark refresh incomplete, retry only with bounded backoff.
Metric unavailableAdd gaps[] entry with requested metric, API, dimensions, date range, and fallback.
Threshold-suppressed breakdownPreserve available aggregate totals; mark dimension breakdown as limited/anonymized.
Private/deleted/unlisted mismatchRecord Data API and Analytics status separately. Do not treat missing public metadata as missing owner analytics.
Reporting API job too newState first report is expected within 48 hours; use Analytics API targeted pull meanwhile.
Backfilled report replaces old reportRe-import newer createTime, archive superseded normalized manifest, update provenance.
Agent attempts to read secretsRefuse and print normalized summary/raw evidence artifact paths only.

Validation Against Current youtube-video-audit

The proposed per-video summary satisfies the existing skill without changes because it is a local JSON file passed via --owner-analytics <path>, includes the exact field families named by the skill, keeps raw/cited evidence separate, and represents missing impressions/CTR, retention, comments, and private metrics as explicit gaps.

$youtube-video-audit https://youtu.be/VIDEO_ID \

Build / No-Build Recommendation

Build the local wrapper, but keep the MVP small.

Recommended MVPDefer
  • OAuth installed-app/browser bootstrap for one owner channel.
  • Analytics API reports.query pulls for core, time series, traffic, device, geography, subscribers, and retention.
  • Data API videos.list and optional commentThreads.list enrichment.
  • Normalized per-video and channel outputs under research/youtube/data/.
  • Raw response retention with field provenance and explicit gaps.
  • Manual and local scheduler-friendly commands.
  • No hosted service, database, GitHub Actions, Studio scraping, screenshots, or write scopes.
  • Reporting API until reach/CTR or durable daily snapshots are required.
  • Monetary/revenue scopes.
  • Multi-channel/multi-user support.
  • Skill rewrites.
  • Hosted dashboards or SaaS behavior.
Reasoning: current repo skills already accept owner analytics as flexible CSV/JSON and tolerate missing private metrics. The highest-value gap is repeatable local normalized evidence, not full Studio parity.
