Audit Goal

Primary question

Do final handoffs after confirmed artifacts reliably tell the user the concrete next step, or do they leave the user with a completed artifact but no actionable continuation route?

Scope boundary

The planned audit is report-only. It may inspect conventions, benchmark fixtures, check scripts, skill contracts, and recent examples, but it must not mutate those surfaces until a follow-up change is explicitly approved.

Preferred durable fix

If evidence proves a gap, prefer a shared convention or benchmark/check guard over skill-local warnings unless the issue is narrow to one skill contract.

Output artifact

The proposed report path is tasks/final-handoff-verification-audit.md. The report should separate verified evidence, inferred risk, responsible contract gap, and recommended follow-up route.

Evidence Surfaces

The audit should start narrow, with the contracts most likely to own final-handoff routing. It should broaden only if those surfaces do not explain the behavior.

Proposed Audit Procedure

Step Work Stop condition
1. Confirm scope Consume approved YAML from this page and restate the target surfaces, report path, and mutation boundary. Any unanswered required gate or unresolved negative feedback keeps the page in review.
2. Read owning contracts Inspect shared alignment, interrogation, routing, shipping, benchmark, and session-triage contracts that mention final handoffs or next steps. If a single contract already clearly owns the behavior, avoid broad repo-wide scanning.
3. Sample evidence Review a small set of recent confirmed artifacts, terminal handoffs, task reviews, and benchmark checks for next-step routing presence or absence. If evidence is inconclusive, report the gap rather than asserting a verified issue.
4. Classify cause Decide whether the problem is agent noncompliance with an adequate contract, missing shared contract language, missing benchmark/check coverage, stale mirrored skill text, or no verified problem. Do not recommend a skill change when evidence points only to one-off noncompliance.
5. Write report Create or update tasks/final-handoff-verification-audit.md with target, issue, verdict, timeline, root cause, responsible gap, recommended fix, validation plan, confidence, and next skill. The report is the only durable output from the audit unless a later approval authorizes remediation.

Decision Framework

Verified contract gap

If shared conventions do not require confirmed-artifact next-step routing, recommend exact shared wording plus a focused validation check.

Verified check gap

If the contract is clear but no benchmark or audit protects it, recommend a check or fixture family that recognizes final handoff route quality.

Skill-local gap

If only one skill has stale or missing final output guidance, route a narrow skill update through the skill-development workflow and preserve versioning rules.

Agent noncompliance

If the existing contract is adequate and the agent ignored it, say so directly and avoid adding more warnings to already adequate docs.

Recommendation rule: the audit may recommend convention or benchmark changes only if the approved gates allow recommendations and the evidence identifies an actual contract or check gap. Recommendation is not implementation approval.
