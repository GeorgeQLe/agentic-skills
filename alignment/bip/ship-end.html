What Shipped

  • Alignment and interrogation skills now point to shared resolver stubs instead of duplicating full convention prose in every skill section.
  • Source checkouts resolve conventions from docs/*-convention.md; packaged installs resolve the same convention text from packaged assets.
  • Existing sibling ALIGNMENT-PAGE.md and INTERROGATION-PAGE.md files remain supported as legacy fallback artifacts.
  • Generators, audits, package command help, documentation, and focused tests were updated to validate resolution semantics.

Source-safety boundary: drafts avoid implying that old installed skills break, avoid inflated quality claims, and do not expose private session details beyond the public repository work.

Fresh-Audience Brief

fresh_audience_context
agentic-skills packages reusable agent skills. Each skill can include page conventions for alignment and interrogation workflows. This migration reduces duplicated convention prose by moving the canonical instructions into shared packaged assets and leaving short skill stubs behind.
jargon_expansion
A resolver is the lookup path an agent uses to find the convention text. A packaged asset is a file shipped with the installable package. A legacy sibling bundle is the old per-skill convention file kept as fallback for already-installed skills.
public_significance
The public lesson is migration design: shared documentation can replace duplicated runtime instructions without forcing a hard break when installed copies still need compatibility.
