This read-only page was generated after shipping packaged alignment and interrogation HTML scaffold support. It proposes source-safe public angles only; it does not publish posts, write social-ledger records, or create approval gates.

What Shipped

  • Packaged starter templates for alignment and interrogation HTML pages under assets/templates/.
  • Strict CLI scaffold commands: gskp alignment pages scaffold and gskp interrogation pages scaffold.
  • Package staging, npm boundary checks, CLI validation tests, and fixture audits proving generated starter pages pass the existing page auditors.
  • create-alignment-page guidance now uses scaffold support for ad hoc pages without making normal skills invoke it as a subskill.

Fresh-Audience Brief

fresh_audience_context
agentic-skills packages reusable agent skills. Some skills produce durable HTML review pages. This update adds starter HTML scaffolds to the installable package so downstream repos can create valid page shells without relying on this source checkout's local docs and scripts.
jargon_expansion
A scaffold is a starter file with valid metadata and placeholders. An alignment page is a review/approval surface for finished work. An interrogation page is a pre-research question surface. A producing skill still owns the real evidence, gates, and YAML compiler.
public-facing significance
The public lesson is packaging boundary design: ship useful starter infrastructure without centralizing domain-specific approval logic or forcing a subskill dependency.
