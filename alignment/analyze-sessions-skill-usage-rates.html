3. Usage Concentration — Head vs. Dead Tail

The distribution is not just skewed — it is a spike plus a long dead tail. Of the 2,698 Claude repo-skill invocations, the top 3 (ship, ship-end, investigate) account for 2,106 (78%).

Invocation tierRepo skillsShare of 182
Never invoked as a command9552%
Invoked exactly once2715%
Invoked 2–5 times2413%
Invoked more than 5 times3620%

Caveat — "unused" ≠ "dead": invocation rate alone can't tell a genuinely dead skill from a new or library skill. The whole game-*, devtool-*, creator-*, most youtube-*, and the research-framework sub-skills sit at 0–2 lifetime invocations. Some may be recently built, built for other repos/users, or invoked indirectly (framework sub-skills are called by a parent skill, not typed). Separating dead from new/library is exactly what a skill-inventory pass would resolve (Rec 2).

4.1 /investigate — the standout gap (~3:1 manual-to-skill)

Every bare "why is X" skips the skill's claim-validation → root-cause → git-history-trace and just gets an ad-hoc answer. This is the largest single shadowed intent and the one where skipping the skill costs the most.

why is onCategoryHover possibly undefined?
why are there version conflicts? Are you referring to between the drizzle orm version used in the pokeapps vs …
investigate, confirm, commit, and then push

4.2 Git / docs / roadmap shipping asks typed as sentences

ok can you push the changes to github with the appropriate commit message?
can you update the documentation and push to github?
update TODO.md
can you pull from master now? (typed 2× → /sync)

Why this is a habit gap, not a skill gap: /ship is muscle memory (1,595 uses) because it's the natural end-of-turn action. The shadowed skills fire from a mid-work realization ("wait, why is this…") where you're already typing a sentence, so the slash prefix never happens. Bare "why is X" also doesn't feel like it needs a skill — but that's exactly /investigate's job.

Count precision: "commit & push" is the tightened count (requires git/github/commit-message/branch context, excludes database/seed pushes). A looser "commit OR push" match returns ~696 but folds in DB-push false positives, so the tightened 211 is the conservative figure. Buckets use first-match keyword classification and are not mutually exhaustive.

5. Claude vs. Codex Personality Split

The two tools are used for different jobs — usage rates reflect that clearly:

  • Claude = the shipping surface. Dominated by ship / ship-end / investigate / sync — day-to-day dev flow.
  • Codex = the skill-development bench. Its top commands are targeted-skill-builder (93), benchmark-test-skill (82), session-triage (65), and benchmark-agent-review (32) — authoring and testing skills, not shipping product.
  • Only 699 of 9,167 Codex prompts (7.6%) are commands vs 23% repo-skill on Claude — Codex work leans on longer freeform plan-implement prompts.

8. Highest-Impact View (avoided manual prompts)

  1. /investigate prompt-nudge hook — ~550 historical hits, the largest single shadowed intent, and the highest-value one (it restores claim-validation before an answer).

Alternatives considered

  • Do nothing — the head is healthy. Rejected as a full answer: true for shipping, but it leaves the ~550 shadowed /investigate asks (the highest-value gap) on the table.
  • Auto-run /investigate on any "why" prompt. Rejected — too aggressive; a suggestion nudge preserves user control and avoids false triggers on rhetorical questions.
  • Delete all 95 unused skills now. Rejected — can't distinguish dead from new/library/indirect without the inventory audit first (Rec 2).
