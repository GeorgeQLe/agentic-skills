For the user flow map, I see it as the start of the wireframe tree which fans out per user flow, ux-variations should expand upon a particular user flow by exploring how users can progress through a flow in different ways, and then for ui-interview to be four steps: 1) investigate a particular ux-variation proposed for a specific user flow and determine what other user-flow it touches, coordinates with other ux-variations or ui-proposals as available, 2) design a proposed ui and display a visual mockup in html 3) interrogate the user for alignment over a UI for the UX variation experiment or if the visual mockup is off-base to try again with feedback 4) approve or reject this branch of the user flow and then route to the next ux-variation or user flow as needed
