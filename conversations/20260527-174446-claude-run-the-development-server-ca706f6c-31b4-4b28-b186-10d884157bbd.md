Found three bugs in `SealedPack.tsx`, all in the same file:

1. **Primary bug**: The tear completion chains `onTear()` → `cardSlideY` animation → `onOpen()`. But `onTear()` triggers a re-render that switches render branches, and the `layoutId` transition kills the in-flight `cardSlideY` animation — so `onOpen()` never fires.
2. **`handleLostPointerCapture`** doesn't check the drag threshold — fast swipes can lose pointer capture and leave the flap stuck.
3. **No spring-back** when you drag below the threshold and release.

The fix decouples `onOpen` from the animation chain by using `onLayoutAnimationComplete` to detect when the layout transition settles, then opens the drawer. Both pointer handlers get extracted into shared `completeTear()`/`revertTear()` helpers.
