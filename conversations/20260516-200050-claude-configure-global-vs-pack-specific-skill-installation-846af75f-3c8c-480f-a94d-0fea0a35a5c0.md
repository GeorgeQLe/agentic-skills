Yes — the install script (`install.sh`) **only installs skills from `global/claude/` and `global/codex/`** into `~/.claude/skills` and `~/.codex/skills` respectively. It explicitly does NOT install pack skills globally. Line 142 even says:

So the architecture is already:

1. **`install.sh`** → symlinks `global/claude/*` and `global/codex/*` to the user-wide skill directories (available in all projects)
2. **`scripts/pack.sh install <pack>`** → installs pack-specific skills into a particular project's `.claude/skills/` or equivalent (project-local only)

The global skills (~30+ in `global/claude/`) are the ones that show up in every session regardless of project. Pack skills stay project-local unless you explicitly symlink them globally.
