- Create `tests/tsconfig.json` targeting ESNext/NodeNext

### 2. Build the harness utilities
- `tests/harness/types.ts` — `RunResult`, `SkillMeta`, `Assertion` types
- `tests/harness/runner.ts` — function that runs `claude --print --permission-mode bypassPermissions --max-budget-usd <cap> -p "<prompt>"` in a temp directory with the pack installed, returns stdout + files written
- `tests/harness/judge.ts` — validators: `hasRequiredSections(md, sections[])`, `validFrontmatter(path)`, `validDAG(dagMd)`
- `tests/harness/fixtures.ts` — reads from `tests/fixtures/` directory

### 3. Write Layer 1 tests (structural, no LLM)
- **frontmatter.test.ts**: Glob all `packs/**/SKILL.md`, parse frontmatter, assert `name`, `description`, `type`, `version` present and valid
- **routing-graph.test.ts**: Extract "Next-Skill Routing" sections, parse skill references, verify they exist in the same pack or globally
- **install.test.ts**: Run `pack.sh install alignment-loop` in a tmp dir, verify symlinks point to correct sources, verify `.agents/project.json` is valid
- **output-paths.test.ts**: Parse "Output" sections for file paths, check no two skills in a pack write to the same path

### 4. Write Layer 2 tests (LLM execution)
- **destination-doc.test.ts**: Install pack in temp dir, seed with `fixtures/inputs/app-idea-fitness-tracker.md` as a grill-me summary context file, run `/destination-doc fitness-tracker`, assert `research/destination-fitness-tracker.md` exists with required sections (Intent, Success Criteria, Taste Notes, Out of Scope, Open Questions)
- **vertical-slice.test.ts**: Install pack in temp dir, copy `fixtures/inputs/destination-doc-sample.md` to `research/`, run `/vertical-slice-splitter research/destination-doc-sample.md`, assert `issues/DAG.md` + at least 2 `issues/0*.md` files exist with required structure
- **full-pipeline.test.ts**: Chain destination-doc → vertical-slice-splitter end-to-end

### 5. Add fixtures
- `tests/fixtures/inputs/app-idea-fitness-tracker.md` — a pre-grilled summary (simulating grill-me output already in context)
- `tests/fixtures/inputs/destination-doc-sample.md` — a known-good destination doc to feed to vertical-slice-splitter
- `tests/fixtures/golden/destination-doc-schema.json` — required section headings
- `tests/fixtures/golden/issue-card-schema.json` — required frontmatter/sections per issue card

### 6. Add npm scripts and root-level integration
- `tests/package.json` scripts: `"test:layer1"`, `"test:layer2"`, `"test"` (both)
- Add a note to the root README or CLAUDE.md about running tests

## Key Design Decisions

- **`claude --print` only** — no SDK calls. Closest to real usage, skills load via symlinks naturally. Multi-turn testing (grill-me) deferred to a future iteration since it needs simulated user input.
- **Temp directories per Layer 2 test** — each test gets a fresh `mkdtemp`, runs `pack.sh install`, executes, then cleans up. No shared state.
- **Budget caps** — each `claude --print` call gets `--max-budget-usd 0.50` to prevent runaways.
- **Layer 2 tests are opt-in** — run via `pnpm test:layer2` explicitly, not in default `pnpm test` (which only runs layer1).
