Close Close Sequence — Phases 10–16

The close flow is a genuine sequential callback chain. PackFlowPhase transitions: drawer-open → closing-collapse → sheet-exiting → layout-morph-out → drop-elevation → sealed.

Critical invariant: activePack survives through phases 13–16 and is only cleared when phase returns to sealed at the end of phase 16. This keeps the SealedPack in its “opened” render branch so the shared layoutId card can morph back.

PackOpener.tsx:102–194
useLayoutEffect(() => {
  if (!isClosing || collapseState) return;
  dbgRef.current.mark("collapse-measure");

  // Walk up to find scroll container
  // Find visible top-left card via bounding rects
  let targetIndex = 0;
  let bestTop = Infinity, bestLeft = Infinity;
  const TOP_TOLERANCE = 10;

  cards.forEach((cardEl, i) => {
    const rect = cardEl.getBoundingClientRect();
    if (rect.bottom < viewport.top || rect.top > viewport.bottom) return;
    if (rect.top < bestTop - TOP_TOLERANCE) {
      bestTop = rect.top; bestLeft = rect.left; targetIndex = i;
    } else if (Math.abs(rect.top - bestTop) <= TOP_TOLERANCE
               && rect.left < bestLeft) {
      bestLeft = rect.left; targetIndex = i;
    }
  });

  const offsets = cards.map((card, i) => {
    if (i === targetIndex) return { x: 0, y: 0 };
    const rect = card.getBoundingClientRect();
    return { x: targetRect.left - rect.left, y: targetRect.top - rect.top };
  });

  setCollapseState({ targetIndex, offsets });
  dbgRef.current.mark("collapse-fan");
}, [isClosing, collapseState, ...]);
  • collapseState null → { targetIndex, offsets[] }
  • collapseCount → 0 (reset)
  • expectedCollapseCount → skills.length - 1

Apex Gates Deep-Dive

Phases 15 and 16 are the two APEX frames — the highest-risk animation boundaries. They run inside a single onLayoutAnimationComplete handler in SealedPack, gated sequentially.

layout-morph-out (Phase 15)

After the sheet exits, PackOpener unmounts and its layoutId card disappears. Framer-motion morphs the card back to SealedPack’s position. When the layout animation completes:

  • The card is at z-index 60, sitting above the pack body at cardSlideY = -180
  • The layout-morph-out gate holds this exact visual frame for inspection in debug mode
  • Guard condition: flowPhase === "layout-morph-out" && !isDrawerOpen && wasInDrawer && !isCloseMorphBackInFlight
  • The one-shot isCloseMorphBackInFlight ref prevents re-entry if framer fires the callback again

On gate release: onCloseMorphComplete() → page sets phase = "drop-elevation"

drop-elevation (Phase 16)

Immediately after the morph-out gate, the drop-elevation gate fires in the same handler:

  • On gate release: wasInDrawer = false, setCardElevated(false) — z-index drops from 60
  • onDropElevationComplete() → page clears activePack and resets phase = "sealed"
  • The descent spring (cardSlideY: -180 → 0, spring 300/25) runs after z-index drops
  • After the spring, hasCardTriggered and isCloseMorphBackInFlight reset so the pack can be re-opened
Why two gates in one handler? The morph-back and elevation-drop are visually distinct moments but share the same onLayoutAnimationComplete callback. Splitting them into two sequential gates lets the debug harness freeze at each visual state independently. In production (non-stepped mode), both gates resolve immediately and the visual transition is a single fluid motion.

Performance Investigation Scope

Non-gated preview of the follow-up investigation after this sequence is confirmed.

Once the 16-phase sequence above is confirmed, the performance investigation will focus on:

  • Close sequence jank: Phases 12–16 involve multiple sequential spring handoffs and layout measurements. Profile whether the collapse → sheet exit → morph-back chain drops frames.
  • Layout thrash in morph-back: Phase 15 sets cardElevated, cardSlideY, and wasInDrawer in a useLayoutEffect. Verify this doesn’t cause unnecessary re-renders or layout recalculations.
  • Fan-out stagger cost: Phase 8 staggers up to 12 cards. Measure whether the stagger delay creates perceptible jank on lower-end devices.
  • activePack retention: The critical invariant (phases 13–16) means the SealedPack re-renders in its opened branch while simultaneously running exit animations. Profile the render cost.
  • Debug overhead: The debugReport calls and motion value subscriptions may add overhead even when the debug panel is closed. Check if debugEnabled guards are sufficient.
