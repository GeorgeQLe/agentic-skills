What Shipped (Source Basis)

The shipped boundary is a workflow-contract change to the session wrap-up skill. Build-In-Public post suggestions generated at the end of a "ship" used to print as inline terminal markdown, while suggestions tied to a review/alignment page already wrote to a single reviewable HTML file. This session unified the two: the wrap/ship batch now writes to one HTML page instead of scrolling past in the terminal.

  • The post-confirmation BIP page trigger was broadened to also fire after a clean ship (for wrap-up skills with no preceding review page), reusing the existing audited page shape — no new file format, audit, or generator.
  • The wrap/ship skill now writes its exhaustive per-channel batch to a single confirmed, read-only HTML page, archives any prior version, links it in a central index, and prints a one-line pointer instead of dumping candidates inline.
  • This very page is the dogfooded output of that change.

All drafts below are source-safe: no secrets, credentials, private repo context, customer names, or unverifiable metrics. The work described is a general workflow/devtool pattern, not a product commitment.
