Interrogation Page Example

This documentation-only example shows the shape an AFPS skill can use before committing to deep research, design, or implementation. A real active page would live under interrogation/ and follow the bundled interrogation-page convention for the producing skill.

Purpose

Confidence gate

Confirm the agent understands the user's goal, available evidence, risk, and unresolved assumptions before creating canonical artifacts.

Open inputs

Ask genuinely answerable questions, not fake confirmations. Every round should reduce uncertainty.

Round 1 · Assumptions manifest

Before asking anything open-ended, the skill states back what it already believes — each row tagged by where the belief came from. These are intake hypotheses, not findings: confirm what holds, correct what's wrong, flag what needs research. Nothing here is canonical.

Open question · recommend and override

At least one input each round must be genuinely open — not a disguised yes/no. The skill may still offer a recommended answer with its confidence, but the user can override freely. This is where new uncertainty actually gets reduced.

Confidence gate · loop exit

The gate is live. It counts every unconfirmed assumption plus every empty required answer. While anything is open, the skill stays in stage zero; when the count reaches zero, it may advance to stage one — the interrogation page never becomes the canonical artifact.

Confidence gate: 4 open items — stay in stage zero 3 assumptions to resolve · 1 required answer empty

Compile responses

This is the round's payoff: the page compiles every assumption decision and open answer into a captured-answers YAML. The user copies it, clears context, and pastes it into a fresh session — a new agent reads the sidecar, runs the confidence gate, and either emits the next round or advances to stage one. That paste-forward is the whole Research Session Loop.
